```text
- 实现 BM25 + Dense Vector 的混合检索流程，使用 RRF 融合多路召回结果，并支持可配置 rerank，对长文档课程资料问答场景下的召回与排序效果进行优化。
- 构建 30+ 条课程问答评测集，使用 Recall@K、MRR、引用命中率对不同 chunk size、top_k 和检索策略进行对比实验，并基于实验结果调整检索参数。
```

## 简历表述注意事项

- 如果没有完成 Docker，就不要在简历里写 Docker Compose。
- 如果 rerank 只是接口未实际跑通，就写“预留 rerank 接口”，不要写“接入 reranker 并提升效果”。
- 如果评测集数量不足 30，就写“小型评测集”，不要写具体数量。
- 不要写“生产级”“高并发”“大规模”这类无法证明的词。
