## README 重点内容

项目背景要说明：

```text
面向课程资料 / 学习资料的智能问答系统，解决资料分散、复习检索效率低、回答缺少引用来源的问题。
```

功能特性要避免空泛，建议写具体能力：

- 支持 PDF / Markdown / TXT 文档导入。
- 支持 chunk 切分和 source metadata 保留。
- 支持 FAISS 向量检索。
- 支持 BM25 + Dense Vector 混合检索。
- 支持 RRF 融合和可选 rerank。
- 支持回答引用来源。
- 支持 FastAPI 调用和 Swagger 调试。
- 支持小型评测集和检索指标对比。

## 验收标准

- 一个陌生面试官看 README 能理解你做了什么。
- README 不只是“安装依赖、运行程序”，而是能体现项目设计和实验结果。
- 项目描述不夸大，不写“生产级大规模部署”之类无法证明的表述。
- README 中有真实问答样例和评测表格。
